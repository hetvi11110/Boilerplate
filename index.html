Document Structure

Best practice: Always include DOCTYPE, html with lang attribute, meta charset, and viewport for responsive design.

<!DOCTYPE html> - Declares HTML5 document type.

<html lang="en"> - Root element with language attribute for accessibility.

<head> - Contains meta information.

<meta charset="UTF-8"> - Sets character encoding.

<meta name="viewport" content="width=device-width, initial-scale=1.0"> - Ensures responsive design.

<title> - Sets page title in browser tab.

<link rel="stylesheet" href="styles.css"> - Links external CSS.

<script src="script.js" defer></script> - Includes external JavaScript.

<base href="https://example.com/"> - Sets base URL for relative links (rarely used).

Headings

Best practice: Use headings hierarchically (h1 first, then h2, etc.) for SEO and accessibility. Only one h1 per page.

Text Formatting

Best practice: Use semantic elements like <strong> and <em> instead of <b> and <i> for better accessibility.

<p>This is a paragraph.</p>

<br> - Line break

<strong>Strong emphasis</strong> - Important text

<em>Emphasized text</em> - Stress emphasis

<b>Bold text</b> - Stylistic bold (use <strong> instead)

<i>Italic text</i> - Stylistic italic (use <em> instead)

<u>Underlined text</u> - Underline

<s>Strikethrough text</s> - Strikethrough

<mark>Marked text</mark> - Highlighted text

H<sub>2</sub>O - Subscript

E=mc<sup>2</sup> - Superscript

<small>Small text</small> - Fine print

<pre>Preformatted text</pre>

<code>console.log('Hello');</code> - Code snippet

Press <kbd>Ctrl+C</kbd> - Keyboard input

<samp>Error 404</samp> - Sample output

<var>x</var> - Variable

<cite>Book Title</cite> - Citation

<q>This is a quote.</q> - Inline quotation

<blockquote>This is a block quotation.</blockquote>

<abbr title="HyperText Markup Language">HTML</abbr> - Abbreviation

<address>123 Main St</address>
